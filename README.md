# Jos-Paulo.aula5-pi

# **Repositório:**

O repositório é o local onde todo o projeto (código-fonte, registro de alterações e arquivos do projeto) está localizado.
Pode estar de maneira local em uma máquina ou remotamente pela nuvem do próprio GitHub.

![image alt](https://github.com/josegomes8-a11y/Jos-Paulo.aula5-pi/blob/4e7755784bf3b249faf60cd99fe9bc6a0743d65f/repositorio-fasouza.png)

# **clone do repositório:**

Ele cria um clone/cópia de todo o conteúdo do repositório remoto e o envia para a máquina local.

# **Branch:**

Ele cria uma cópia do repositório original com o intuito de fazer alterações no projeto, mas sem comprometer o projeto principal.

![image alt](https://github.com/josegomes8-a11y/Jos-Paulo.aula5-pi/blob/00f10ef658d68235a42b4982532b2b42ae5abd84/branch.png)

# **Commits:**

Ele serve para catalogar e informar as alterações do projeto utilizando nomenclaturas dependendo da alteração.
EX: feat: adiciona uma nova funcionalidade, fix: corrige algum erro, docs: atualiza README.

![image alt](https://github.com/josegomes8-a11y/Jos-Paulo.aula5-pi/blob/00f10ef658d68235a42b4982532b2b42ae5abd84/commits.webp)

# **Envio das alterações (push):**

Compartilha o trabalho da máquina local para a nuvem do GitHub, atualizando o projeto diretamente no site.
OBS: Antes de fazer push, é necessário fazer commit!

# **Atualização do repositório local (pull):**

Ele puxa do repositório remoto (nuvem) os dados atualizados para a máquina local.

![image alt](https://github.com/josegomes8-a11y/Jos-Paulo.aula5-pi/blob/00f10ef658d68235a42b4982532b2b42ae5abd84/push%20and%20pull.png)

# **Criação de Issues**

Ele serve como ferramenta de acompanhamento, indicando quando queremos descrever os problemas que temos nos projetos para que sejam resolvidos por qualquer pessoa que quiser contribuir.

![image alt](https://github.com/josegomes8-a11y/Jos-Paulo.aula5-pi/blob/00f10ef658d68235a42b4982532b2b42ae5abd84/issue.jpg)

# **Organização com Milestone**

Ela é um marco que agrupa issues relacionadas, funcionando quase como um filtro, permitindo acompanhar o progresso e auxiliando no controle de prazos.

![image alt](https://github.com/josegomes8-a11y/Jos-Paulo.aula5-pi/blob/a8368954abcf4d1bbdf10fe3cd944bdc9303f832/milestone.png)

# **Gerenciamento com Project**

É uma ferramenta que se integra aos issues e pull requests, ajudando na organização geral do projeto em equipes e informando de forma visual como está o andamento dos projetos.

# **Abertura de Pull Request**

Ele serve para solicitar a integração das alterações feitas em uma branch ao repositório principal.

# **Realização de Merge**

Ele integra uma branch à branch principal, aplicando oficialmente as alterações feitas.

![image alt](https://github.com/josegomes8-a11y/Jos-Paulo.aula5-pi/blob/a8368954abcf4d1bbdf10fe3cd944bdc9303f832/pull%20request%20and%20branch.png)
